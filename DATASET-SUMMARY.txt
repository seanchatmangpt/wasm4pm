EXECUTIVE SUMMARY
===================================================================================

A comprehensive collection of 25+ publicly available, real-world process mining
datasets has been compiled for benchmarking and validating the wasm4pm library.
All datasets are:

  ✓ Publicly accessible (free download, no login required)
  ✓ License-free or CC0/CC-BY (no restrictions on use)
  ✓ Real organizational data or validated synthetic data
  ✓ Well-documented with metadata
  ✓ Available in standard formats (XES, CSV, OCEL)
  ✓ Ranging from 1K to 1.6M events for comprehensive testing

===================================================================================
RECOMMENDED TESTING STRATEGY FOR WASM4PM
===================================================================================

PHASE 1: FAST ITERATION (Use Sepsis Cases)
  Goal: Rapid algorithm prototyping and UI testing
  Dataset: Sepsis (1K cases, 15K events, ~15ms for DFG)
  Metrics: Algorithm correctness, UI responsiveness
  Time per test: < 100ms

PHASE 2: MEDIUM-SCALE TESTING (Use BPIC 2013 & 2020)
  Goal: Algorithm validation on realistic data
  Datasets: BPIC 2013 Incidents, BPIC 2020 Travel Permits
  Metrics: Discovery accuracy, conformance checking
  Time per test: 100-500ms

PHASE 3: PRODUCTION BENCHMARKING (Use BPIC 2015 & Road Traffic)
  Goal: Performance baseline on government/large processes
  Datasets: BPIC 2015 Building Permits, Road Traffic Fine
  Metrics: Scalability to 150K+ cases, memory footprint
  Time per test: 500-3000ms

PHASE 4: EXTREME STRESS TESTING (Use BPIC 2019 & Synthetic 900K)
  Goal: Validate scalability and memory management
  Datasets: BPIC 2019 (1.6M events), Artificial Log (900K events)
  Metrics: Memory usage, algorithm timeout handling, chunking efficiency
  Time per test: 5000-30000ms (may require Web Workers)

PHASE 5: ALGORITHM VALIDATION (Use Process Discovery Contest 2022)
  Goal: Accuracy measurement with known ground truth
  Datasets: PDC 2022 (96 test logs with Petri net ground truth)
  Metrics: Fitness score, precision, recall vs. reference models

BENCHMARK COMMANDS FOR WASM4PM
===================================================================================

# Quick baseline (< 1 second)
npm run bench -- --dataset=sepsis

# Medium benchmarks (2-10 seconds)
npm run bench -- --dataset=bpic2013,bpic2020

# Full benchmark suite (30-60 seconds)
npm run bench -- --dataset=all

# Scalability test (60+ seconds)
npm run bench -- --dataset=bpic2019 --memory-profile

# Compare with baseline
npm run bench -- --compare-baseline

# Generate HTML report
npm run bench -- --output=benchmark_report.html
